My DevSecOps Plan

This is the **$0 Budget Plan**. It focuses *only* on the skills you can learn for **free** using your Debian VM and free online tools. This plan is designed to build the specific, high-value skills that make you "irreplaceable" and get you your first job.

Why am I doing this?

Your diagnostic showed this is your **biggest weakness**. All servers, automation, and containers are **headless** (no GUI). You must be able to operate in a terminal to do *any* part of this job. This phase covers the "Basic Linux commands" and "Terminal emulator" skills.

What:

Mastering core Git and Linux commands until they are muscle memory.

How to Practice ($0):

  • Git: Create a new GitHub repo. `git clone` it. Create a `test.txt` file. `git add .`. `git commit -m "test"`. `git push`. **Do this 5 times.**
  • Linux: On your Debian VM, use **only** the terminal for one day. Navigate: `cd /var/log`, `pwd`. List: `ls -la`. Read: `cat auth.log`. Search: `grep "Failed" auth.log`
  • Resource: OverTheWire: Bandit

✅ Definition of Done: You can push a file to GitHub and find a line in a log file using `grep` without looking up the commands.

What:

Learning to be a *server administrator*, not just a *desktop user*. This is the "Monitoring" and "IDS/IPS" foundation.

How to Practice ($0):

  • Service Mgmt: Install web server: `sudo apt install nginx`. Check status: `sudo systemctl status nginx`. Stop: `sudo systemctl stop nginx`
  • Log Analysis: Go to `/var/log/`. Find hacker IPs: `grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr`. Watch live: `sudo tail -f /var/log/syslog`
  • Resource: LetsDefend.io

✅ Definition of Done: You can start/stop `nginx` and build the `grep` command to find the top 10 attacking IPs.

Why am I doing this?

This gives you **authority**. You're re-learning to hack with a new goal: to **write code that hacks for you**. This covers the "Coding skills (Python)" and "Security concepts" requirements.

What:

Mastering the "big 4" OWASP vulnerabilities that all scanners look for.

  • Learn: SQL Injection, XSS, OS Command Injection, IDOR.
  • Where: PortSwigger Web Security Academy (FREE).

✅ Definition of Done: You have completed the **Practitioner**-level labs for SQLi and XSS.

What:

Writing Python scripts that act like tools. This is the foundation for all "Automation".

  • Learn: `requests` library (for HTTP) and `argparse` (for CLI arguments).
  • Where: Free guides online. You don't need a book.

How to Practice ($0):

Write a Python script (`scanner.py`) that takes a URL as an argument (`-u http://...`) and prints all the links.

✅ Definition of Done: You have a working **Python script** that uses `requests` and `argparse`.

Why am I doing this?

This is the "Cloud-Native" skill. You need to package your code so it runs the same way for everyone, everywhere. **Docker is the foundation for 90% of all "Cloud" skills.**

What:

Writing a "recipe" (`Dockerfile`) that packages your app.

  • Learn: `Dockerfile` syntax (`FROM`, `RUN`, `COPY`, `CMD`), `docker build`, `docker run`.
  • Where: Docker's official "Get Started" guide (FREE).

How to Practice ($0):

Write a `Dockerfile` for your Phase 1 script. Build it (`docker build . -t my-scanner`) and run it.

✅ Definition of Done: Your Phase 1 script runs successfully from a **Docker container** you built.

Why am I doing this?

This is the core "DevSecOps" skill. You are building the automated assembly line that will act as your 24/7 security guard. This is where you implement SAST, DAST, and **SCA**.

What:

Building your first workflow in **GitHub Actions** and adding **SAST** (Static Analysis).

  • Learn: **YAML** syntax for GitHub Actions (`.github/workflows/main.yml`).
  • Tool: **Semgrep** (FREE SAST tool).

How to Practice ($0):

Commit a hardcoded password. Add Semgrep Action. Your pipeline must turn **RED**.

✅ Definition of Done: Your pipeline fails when you push a hardcoded secret.

What:

Adding **SCA** to find vulnerable libraries and **DAST** to attack the running app.

  • SCA Tool: **Trivy** (FREE).
  • DAST Tool: **OWASP ZAP** (FREE).

How to Practice ($0):

  • Add Trivy step to find vulnerable Python libraries.
  • Add OWASP ZAP Baseline Scan against a test URL.

✅ Definition of Done: One pipeline running **SAST**, **SCA**, and **DAST** on every push.

Why am I doing this?

This project **proves** you are not just another DevSecOps guy. Combine all skills to show you're an engineer, a hacker, and an automator.

What:

A pipeline that uses your **own exploit script** as a security test.

  • Step 1: Build a Flask app **deliberately vulnerable** to SQL Injection.
  • Step 2: Write `exploit.py` that attacks and proves the SQLi exists.
  • Step 3: Write `docker-compose.yml` that runs your app.
  • Step 4: Build GitHub Action: start app → wait 10s → run exploit → FAIL if exploit succeeds.

✅ Definition of Done: Push vulnerable code → pipeline turns **RED** (fails). Push patched code → turns **GREEN** (passes). Put on resume.
